# 🌿 Plant Disease Detector — AI Web App

A deep learning web application that detects plant diseases from leaf images using a **MobileNetV2 CNN** trained with transfer learning on the PlantVillage dataset (38 disease classes, 87,000+ images).

**Built with:** Python · TensorFlow/Keras · Flask · HTML/CSS/JS  
**Author:** Abdallah Elbarbary · MIU Computer Science — AI

## 🏋️ Train the Model

```bash
python train.py
```

**What happens:**
- Phase 1 (10 epochs): Trains only the classification head on top of frozen MobileNetV2
- Phase 2 (10 epochs): Fine-tunes the top 30 layers of MobileNetV2 with a very low learning rate
- Saves best model to `model/plant_disease_model.h5`
- Saves class names to `model/class_names.json`

**Expected results:** ~95%+ validation accuracy (MobileNetV2 transfer learning is very strong on this dataset)

**Training time:**
- GPU: ~30–45 minutes
- CPU: ~3–4 hours (still works, just slower)

> 💡 **Tip:** Run this on Google Colab (free GPU) if you don't have a GPU locally. Upload the saved model back when done.

## 🧠 Model Architecture

```
Input (224×224×3)
    ↓
MobileNetV2 (pretrained on ImageNet, top removed)
    ↓ [base frozen in Phase 1, top 30 layers unfrozen in Phase 2]
GlobalAveragePooling2D
    ↓
BatchNormalization
    ↓
Dense(256, relu)
    ↓
Dropout(0.4)
    ↓
Dense(38, softmax)  ← output: probability per disease class
```

**Why MobileNetV2?**
- Pretrained on 1.2M ImageNet images → strong general visual features
- Lightweight (3.4M params) → fast inference, deployable on free hosting
- Transfer learning = ~95% accuracy with only ~30 min training
